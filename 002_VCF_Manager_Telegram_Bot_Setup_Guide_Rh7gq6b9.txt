## 1. Prerequisites

### Python 3.10+

You need Python **3.10 or newer**. To check your current version:

```bash
python --version
# or
python3 --version
```

If you don't have Python installed, download it from [https://www.python.org/downloads/](https://www.python.org/downloads/).

> **Windows users:** During installation, check the box **"Add Python to PATH"** before clicking Install.

---

### Tesseract OCR

Tesseract is required for reading text from images (OCR). Install it for your operating system:

#### Windows

1. Download the installer from:
   [https://github.com/UB-Mannheim/tesseract/wiki](https://github.com/UB-Mannheim/tesseract/wiki)
2. Run the installer and **note the installation path** (default: `C:\Program Files\Tesseract-OCR\tesseract.exe`)
3. *(Optional)* Add Tesseract to your system PATH:
   - Open **Start → Search "Environment Variables"**
   - Under **System Variables**, select `Path` → **Edit**
   - Click **New** and paste: `C:\Program Files\Tesseract-OCR`
   - Click OK to save

#### Linux (Ubuntu / Debian)

```bash
sudo apt update
sudo apt install tesseract-ocr -y
```

#### Linux (Fedora / CentOS / RHEL)

```bash
sudo dnf install tesseract -y
# or for older systems:
sudo yum install tesseract -y
```

#### macOS

```bash
brew install tesseract
```

> If you don't have Homebrew: `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`

#### Verify Tesseract is installed

```bash
tesseract --version
```

You should see output like `tesseract 5.x.x`.

## 2. Bot Creation with BotFather

### Step-by-step

1. Open Telegram and search for **@BotFather** (blue checkmark ✅).
2. Start a chat and send:
   ```
   /newbot
   ```
3. BotFather will ask: **"What's the name of your bot?"**
   Enter a display name, e.g.:
   ```
   VCF Manager Bot
   ```
4. BotFather will ask: **"What username would you like to give it?"**
   Enter a unique username ending in `bot`, e.g.:
   ```
   myvcfmanager_bot
   ```
5. BotFather will reply with a message containing your **bot token**, which looks like:
   ```
   1234567890:ABCDefGhIJKlmNoPQRsTUVwxYZ-abcdefghi
   ```
6. **Copy this token and keep it secret.** You'll need it in the next section.

> **Tip:** You can also set a description and profile photo for your bot using `/setdescription` and `/setuserpic` in BotFather.

## 3. Installation

### Step 1 — Download the project

**Option A — Clone with Git:**
```bash
git clone https://github.com/your-username/vcf-manager-bot.git
cd vcf-manager-bot
```

**Option B — Download ZIP:**
- Click the green **Code → Download ZIP** button on the repository page
- Extract the ZIP and open a terminal inside the extracted folder

---

### Step 2 — Create a virtual environment (recommended)

```bash
# Create virtual environment
python -m venv venv

# Activate it — Windows:
venv\Scripts\activate

# Activate it — macOS/Linux:
source venv/bin/activate
```

You'll see `(venv)` at the start of your terminal prompt when it's active.

---

### Step 3 — Install dependencies

```bash
pip install -r requirements.txt
```

This installs all required packages:

| Package | Purpose |
|---|---|
| `python-telegram-bot` | Telegram Bot API wrapper |
| `phonenumbers` | Phone number parsing & validation |
| `openpyxl` | Read/write Excel files (.xlsx) |
| `pytesseract` | Python wrapper for Tesseract OCR |
| `Pillow` | Image processing |
| `python-dateutil` | Date parsing utilities |
| `pytz` | Timezone support |

---

### Step 4 — Configure Tesseract path (Windows only)

On Windows you must tell `pytesseract` where Tesseract is installed. Open `bot.py` and find this line:

```python
# pytesseract.pytesseract.tesseract_cmd = r'C:\Program Files\Tesseract-OCR\tesseract.exe'
```

Uncomment it (remove the `#`) and update the path if you installed Tesseract elsewhere:

```python
pytesseract.pytesseract.tesseract_cmd = r'C:\Program Files\Tesseract-OCR\tesseract.exe'
```

> **macOS/Linux users:** This line is not needed — Tesseract is found automatically via PATH.

## 4. Configuration

### Bot Token

Open `bot.py` in any text editor (Notepad, VS Code, nano, etc.) and find this line near the top:

```python
BOT_TOKEN = "YOUR_BOT_TOKEN_HERE"
```

Replace `YOUR_BOT_TOKEN_HERE` with the token you copied from BotFather:

```python
BOT_TOKEN = "1234567890:ABCDefGhIJKlmNoPQRsTUVwxYZ-abcdefghi"
```

> **Security tip:** Never share your token publicly or commit it to a public repository. If your token is leaked, regenerate it with BotFather using `/revoke`.

---

### Timezone Settings

If your bot displays times and you want them in your local timezone, find the timezone setting in `bot.py`:

```python
TIMEZONE = "UTC"
```

Change `"UTC"` to your timezone string. Examples:

```python
TIMEZONE = "America/New_York"     # US Eastern
TIMEZONE = "America/Los_Angeles"  # US Pacific
TIMEZONE = "Europe/London"        # UK
TIMEZONE = "Europe/Berlin"        # Central Europe
TIMEZONE = "Asia/Dubai"           # Gulf Standard Time
TIMEZONE = "Asia/Kolkata"         # India
TIMEZONE = "Asia/Tokyo"           # Japan
```

A full list of timezone names: [https://en.wikipedia.org/wiki/List_of_tz_database_time_zones](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones)

## 5. Running the Bot

### Start the bot

Make sure your virtual environment is active, then run:

```bash
python bot.py
```

You should see output like:
```
Bot started successfully...
Polling for updates...
```

Open Telegram, find your bot by username, send `/start` — it should respond! 🎉

Press `Ctrl + C` to stop the bot.

---

### Keep the bot running (so it stays alive after you close the terminal)

#### Option A — `screen` (Linux/macOS, simplest)

```bash
# Install screen (if not already installed)
sudo apt install screen -y   # Ubuntu/Debian

# Start a new named session
screen -S vcfbot

# Run the bot inside the screen session
python bot.py

# Detach from screen (bot keeps running): press
Ctrl + A, then D

# To reattach later:
screen -r vcfbot
```

#### Option B — `nohup` (Linux/macOS, quick one-liner)

```bash
nohup python bot.py > bot.log 2>&1 &
```

- Logs are saved to `bot.log`
- To stop it: `kill $(pgrep -f bot.py)`

#### Option C — `systemd` service (Linux, production-grade)

1. Create a service file:
   ```bash
   sudo nano /etc/systemd/system/vcfbot.service
   ```

2. Paste the following (update paths to match your setup):
   ```ini
   [Unit]
   Description=VCF Manager Telegram Bot
   After=network.target

   [Service]
   Type=simple
   User=YOUR_LINUX_USERNAME
   WorkingDirectory=/home/YOUR_LINUX_USERNAME/vcf-manager-bot
   ExecStart=/home/YOUR_LINUX_USERNAME/vcf-manager-bot/venv/bin/python bot.py
   Restart=on-failure
   RestartSec=5

   [Install]
   WantedBy=multi-user.target
   ```

3. Enable and start the service:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable vcfbot
   sudo systemctl start vcfbot
   ```

4. Check status:
   ```bash
   sudo systemctl status vcfbot
   ```

5. View logs:
   ```bash
   journalctl -u vcfbot -f
   ```

## 6. Hosting Options

### Option A — VPS (DigitalOcean, AWS, Linode, etc.)

Best for: **Full control, reliable uptime**

1. Create a VPS (Ubuntu 22.04 recommended, 1 GB RAM is enough)
2. SSH into the server:
   ```bash
   ssh root@YOUR_SERVER_IP
   ```
3. Follow the [Installation](#3-installation) steps above
4. Use the `systemd` method to keep the bot running 24/7
5. **Cost:** ~$4–6/month (DigitalOcean Droplet, AWS Lightsail)

---

### Option B — Railway.app (Free tier available)

Best for: **Easiest deployment, no server management**

1. Sign up at [https://railway.app](https://railway.app)
2. Click **New Project → Deploy from GitHub Repo**
3. Connect your GitHub account and select the bot repository
4. In **Variables**, add:
   - `BOT_TOKEN` = your token
   - `TIMEZONE` = your timezone
5. Railway will automatically detect Python and install `requirements.txt`
6. Add a `Procfile` to your repo with:
   ```
   worker: python bot.py
   ```
7. Deploy! Railway keeps the bot running automatically.
8. **Cost:** Free tier available; $5/month for more resources

---

### Option C — Heroku

Best for: **Familiar platform with add-ons ecosystem**

1. Sign up at [https://heroku.com](https://heroku.com) and install the [Heroku CLI](https://devcenter.heroku.com/articles/heroku-cli)
2. Login and create an app:
   ```bash
   heroku login
   heroku create my-vcf-bot
   ```
3. Add Tesseract buildpack (required for OCR):
   ```bash
   heroku buildpacks:add heroku/python
   heroku buildpacks:add https://github.com/heroku/heroku-buildpack-apt
   ```
4. Create an `Aptfile` in your project root:
   ```
   tesseract-ocr
   ```
5. Create a `Procfile`:
   ```
   worker: python bot.py
   ```
6. Set your bot token:
   ```bash
   heroku config:set BOT_TOKEN="your_token_here"
   ```
7. Deploy:
   ```bash
   git push heroku main
   heroku ps:scale worker=1
   ```
8. **Cost:** Free tier removed; ~$7/month for Eco dynos

---

### Option D — PythonAnywhere

Best for: **Python-focused, beginner-friendly dashboard**

1. Sign up at [https://www.pythonanywhere.com](https://www.pythonanywhere.com)
2. Open a **Bash console**
3. Clone your repo and install dependencies (same steps as [Installation](#3-installation))
4. Go to **Tasks** tab and add a scheduled task:
   ```bash
   /home/YOUR_USERNAME/vcf-manager-bot/venv/bin/python /home/YOUR_USERNAME/vcf-manager-bot/bot.py
   ```
5. **Note:** Free tier has outbound network restrictions. Upgrade to a paid plan (~$5/month) for reliable Telegram API access.

## 7. Troubleshooting

### ❌ `ModuleNotFoundError: No module named 'telegram'`

**Cause:** Dependencies not installed or virtual environment not active.

**Fix:**
```bash
# Activate virtual environment first
source venv/bin/activate        # macOS/Linux
venv\Scripts\activate           # Windows

# Then install dependencies
pip install -r requirements.txt
```

---

### ❌ `TesseractNotFoundError` / `tesseract is not installed or not in your PATH`

**Fix on Windows** — Add the path in `bot.py`:
```python
pytesseract.pytesseract.tesseract_cmd = r'C:\Program Files\Tesseract-OCR\tesseract.exe'
```

**Fix on Linux:**
```bash
sudo apt install tesseract-ocr -y
```

**Fix on macOS:**
```bash
brew install tesseract
```

**Verify it works:**
```bash
tesseract --version
```

---

### ❌ `Unauthorized` / `Invalid token`

**Cause:** The bot token in `bot.py` is wrong or has extra spaces.

**Fix:**
- Double-check the token you copied from BotFather
- Make sure there are no spaces before or after the token string
- If needed, generate a new token: open BotFather → select your bot → `/revoke`

---

### ❌ `ConnectionError` / `Network unreachable`

**Cause:** No internet connection, or Telegram is blocked in your region.

**Fix:**
```bash
# Test connectivity
curl https://api.telegram.org

# If blocked, use a proxy or VPN, or host on a VPS in an unrestricted region
```

---

### ❌ `PermissionError` when writing files

**Cause:** The bot doesn't have write access to its working directory.

**Fix on Linux:**
```bash
# Give ownership to your user
sudo chown -R $USER:$USER /path/to/vcf-manager-bot

# Or set write permissions
chmod -R 755 /path/to/vcf-manager-bot
```

---

### ❌ Bot stops responding after some time

**Cause:** The process was terminated (terminal closed, server rebooted).

**Fix:** Use one of the persistent methods from [Section 5](#5-running-the-bot) — `screen`, `nohup`, or `systemd`.

---

### ❌ `pip: command not found`

**Fix:**
```bash
# Try pip3 instead
pip3 install -r requirements.txt

# Or use the Python module directly
python -m pip install -r requirements.txt
python3 -m pip install -r requirements.txt
```

---

### ❌ `python: command not found`

**Fix:**
```bash
# Try python3 instead
python3 bot.py
```

> On many Linux/macOS systems, `python3` is the correct command. You can create an alias: `alias python=python3`
