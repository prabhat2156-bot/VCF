# WhatsApp Group Manager Bot — Complete Setup Guide

---

## 📁 Project Structure

```
whatsapp-bot/
├── bot.py              ← Python Telegram Bot (main)
├── bridge.js           ← Node.js Baileys Bridge Server
├── .env                ← Environment variables (create this)
├── requirements.txt    ← Python dependencies
├── package.json        ← Node.js dependencies
├── sessions/           ← Python temp session data (auto-created)
├── wa_sessions/        ← WhatsApp Baileys sessions (auto-created)
└── bot.log / bridge.log ← Log files
```

---

## 🔧 Step 1: Prerequisites

- Python 3.10+ installed
- Node.js 18+ installed
- A Telegram Bot Token from [@BotFather](https://t.me/BotFather)
- A phone with WhatsApp installed (for connecting)

---

## 📦 Step 2: Install Python Dependencies

```bash
pip install python-telegram-bot==20.7 aiohttp python-dotenv aiofiles
```

**requirements.txt:**
```
python-telegram-bot==20.7
aiohttp>=3.9.0
python-dotenv>=1.0.0
aiofiles>=23.2.1
```

---

## 📦 Step 3: Install Node.js Dependencies

```bash
# Navigate to the project folder
mkdir whatsapp-bot && cd whatsapp-bot

# Place bridge.js here, then run:
npm init -y
npm install @whiskeysockets/baileys @hapi/boom pino express qrcode
```

**package.json dependencies section:**
```json
{
  "dependencies": {
    "@whiskeysockets/baileys": "^6.7.9",
    "@hapi/boom": "^10.0.1",
    "pino": "^8.17.2",
    "express": "^4.18.2",
    "qrcode": "^1.5.3"
  }
}
```

---

## 🔑 Step 4: Create .env File

Create a file named `.env` and add the following content:

```env
# Telegram Bot Token (get from BotFather)
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrSTUVwxyz

# Bridge Server URL (default: localhost:3000)
BRIDGE_URL=http://localhost:3000

# Bridge Secret Key (any random string)
BRIDGE_SECRET=my_super_secret_key_2024

# Your Telegram User ID(s) - comma separated (optional)
ADMIN_IDS=123456789

# Bridge Server Port (for bridge.js)
BRIDGE_PORT=3000
```

> **How to get your Telegram User ID?** Send `/start` to [@userinfobot](https://t.me/userinfobot)

## 📱 Step 7: Connect WhatsApp

1. Send `/start` to the bot
2. Press the **Connect WhatsApp** button
3. Choose **Phone Number** or **QR Code**

### Phone Number Method (Recommended):
1. Enter your phone number (e.g., `+919876543210`)
2. The bridge will generate an 8-digit pairing code
3. Open WhatsApp → Settings → Linked Devices → Link a Device → Link with Phone Number
4. Enter the code
5. ✅ Connected!

### QR Code Method:
1. Choose the QR Code option
2. The bot will send you a QR code image
3. Open WhatsApp → Settings → Linked Devices → Link a Device
4. Scan the QR code
5. ✅ Connected!

## ⚡ Multiple Accounts

Each time you use **Connect WhatsApp**, a new account is added.
The bot automatically manages all accounts, and will prompt you to choose an account whenever multiple accounts are available.

## 📌 VCF File Format

The bot supports `.vcf` (vCard) files. Standard format:

```
BEGIN:VCARD
VERSION:3.0
FN:John Doe
TEL;TYPE=CELL:+919876543210
END:VCARD
BEGIN:VCARD
VERSION:3.0
FN:Jane Doe
TEL;TYPE=CELL:+918765432109
END:VCARD
```

## 🛡️ Security Notes

1. **Never share** your `.env` file — add it to `.gitignore`
2. Keep `BRIDGE_SECRET` as a strong random string
3. Do not expose the bridge server on the public internet
4. Restrict bot access to allowed users only (via `ADMIN_IDS`)
5. Keep a backup of the `wa_sessions/` folder — your sessions are stored there

## 🐛 Troubleshooting

| Error | Solution |
|-------|----------|
| Bridge server not running | Start it with `node bridge.js` |
| `BOT_TOKEN` not set | Add the token to your `.env` file |
| QR code expired | Try Connect WhatsApp again |
| `Account not connected` | Restart the bridge and check the session |
| Members not being added | Check if the number is registered on WhatsApp |

## 🔄 Auto-Restart with PM2 (Production)

```bash
npm install -g pm2

# Start the bridge
pm2 start bridge.js --name "wa-bridge"

# Start the Python bot
pm2 start bot.py --interpreter python3 --name "wa-bot"

# Auto-restart on reboot
pm2 startup
pm2 save
```
